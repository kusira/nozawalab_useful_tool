# param_affine_tool

NPY / RAW / PNG / JPEG 画像を読み込み、トリミング・リサイズ・画像処理を行い、face-alignment（FA）と dlib で顔特徴点を算出・アフィン変換する GUI ツールです。

**Version: 1.0**

---

## 起動方法

### ダブルクリック起動（推奨）

`main.py` と同じフォルダにある **`startup_param_affine_tool.vbs`** をダブルクリックしてください。  
コマンドプロンプトを開かずに GUI が起動します。

### コマンドラインから起動

```bash
python main.py
```

プロジェクトルート（`main.py` があるディレクトリ）で実行してください。

## 対応ファイル形式

| 形式 | 説明 |
|------|------|
| `.npy` | NumPy 配列ファイル |
| `.png` | PNG 画像 |
| `.jpg` / `.jpeg` | JPEG 画像 |
| `.raw` | RAW バイナリ（読み込み時に幅・高さ・チャンネル・データ型などを指定） |

## 画面構成

### プレビュー（右側 2×2）

| パネル | 内容 |
|--------|------|
| 元画像 | 読み込んだ画像全体（トリミング前） |
| 画像処理後 | トリミング → リサイズ → 画像処理を適用した結果 |
| アフィン (face-alignment) | FA の特徴点でアフィン変換した結果（**256 × 256**） |
| アフィン (dlib) | dlib の特徴点でアフィン変換した結果（**256 × 256**） |

各パネルのタイトルに解像度（幅 × 高さ）が表示されます。アフィン変換の出力サイズは **256 × 256** です。

### ツールバー（上部）

| ボタン / 操作 | 説明 |
|---------------|------|
| ファイルを開く | 画像ファイルを読み込む |
| RAW再読み込み | RAW ファイルの読み込みパラメータを変更して再読み込み |
| FA / dlib チェック | 特徴点算出で使うエンジンを個別に ON/OFF |
| 特徴点算出 | 有効なエンジンで特徴点検出とアフィン変換を実行 |
| プレビュー保存 | 各段階の画像を PNG で一括保存 |
| 全設定リセット | スライダー・トリミング・特徴点結果をすべて初期化 |

特徴点算出後、処理時間（FA / dlib / YOLO / 合計）がツールバー右側に表示されます。

## 基本的な使い方

### 1. 画像を開く

「**ファイルを開く**」から NPY / PNG / JPEG / RAW を選択します。  
RAW の場合はダイアログで幅・高さ・チャンネル数・オフセット・データ型を入力します。

### 2. 前処理

左パネル「**前処理**」で以下の操作を行います。

#### 反転
- **左右反転** / **上下反転** … 元画像を反転（反転後はトリミング結果がクリアされます）

#### トリミング
1. **範囲選択** … 元画像上でドラッグして矩形を指定
2. **適用** … 切り抜き結果を「画像処理後」パネルへ送る
3. **取消** … 範囲選択モードを中止
4. **リセット** … 適用済みのトリミングを解除

#### リサイズ
- **リサイズ** スライダー（0.1〜1.0）… トリミング後の画像に対する縮尺。1.0 が原寸です。

#### YOLO bbox 倍率
- **YOLO bbox** スライダー（1.0〜3.0）… 顔検出 bbox を中心基準で拡大。1.0 が原寸です。特徴点算出時に反映されます。

### 3. 画像処理

「**画像処理**」のスライダーで明るさ・コントラスト・ガンマ・ぼかし・シャープ・二値化・回転・クリップ・ヒストグラム均等化・反転などを調整します。  
変更はリアルタイムで「画像処理後」プレビューに反映されます（トリミング適用後）。

### 4. 特徴点算出

1. トリミングを適用した状態にする
2. 必要に応じて FA / dlib のチェックを切り替える
3. **特徴点算出** をクリック

- FA の特徴点は **緑**、dlib は **橙** で重ね表示されます
- 各エンジンのアフィン変換結果（**256 × 256**）が下段プレビューに表示されます
- 初回のみモデル読み込みのため「準備中...」と表示されます（処理時間には含まれません）

GPU が利用可能な場合、face-alignment と YOLO は GPU で動作します。dlib のランドマーク推定は CPU です。

### 5. プレビュー保存

「**プレビュー保存**」で保存先フォルダを選ぶと、以下の PNG が出力されます。

| ファイル名 | 内容 |
|------------|------|
| `{名前}_00_original.png` | 元画像（トリミング前） |
| `{名前}_00_cropped.png` | トリミング後（未リサイズ） |
| `{名前}_01_processed.png` | 画像処理後 |
| `{名前}_02_processed_landmarks.png` | 特徴点オーバーレイ付き |
| `{名前}_03_affine_face_alignment.png` | FA アフィン変換（256 × 256） |
| `{名前}_04_affine_dlib.png` | dlib アフィン変換（256 × 256） |

未生成の画像はスキップされます。

## 処理の流れ（まとめ）

```
ファイル読み込み
  ↓
元画像（反転・トリミング範囲指定）
  ↓ 適用
トリミング結果 → 画像処理後パネル
  ↓ リサイズ
  ↓ 画像処理スライダー
  ↓ 特徴点算出（YOLO bbox 倍率 → FA / dlib）
アフィン変換プレビュー
```

## 依存関係（目安）

- Python 3.10+
- OpenCV (`opencv-python`)
- NumPy
- Pillow
- PyTorch + face-alignment（FA 用）
- dlib + `shape_predictor_68_face_landmarks.dat`（dlib 用）
- ultralytics + `yolov8n-face-lindevs.pt`（YOLO 顔検出用）

モデルファイル（`.dat` / `.pt`）は `module/` ディレクトリに配置してください。

---

## 注意事項

- 特徴点算出には、先にトリミングの **適用** が必要です
- **全設定リセット** はスライダー・トリミング・特徴点結果を戻しますが、ファイルの再読み込みは行いません
- 反転を行うとトリミング結果はクリアされます
- RAW 以外では「RAW再読み込み」ボタンは無効です
